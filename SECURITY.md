# Security Policy

## Supported Versions

GMA is still evolving quickly. Security fixes are made on `dev` and included in the next tagged release. Only the latest `2.x` release is supported; `1.x` and older snapshots are not maintained.

| Version | Supported |
| --- | --- |
| `dev` | Pre-release security fixes |
| Latest `2.x` release | Yes |
| `1.x` and older | No |

## Report a Vulnerability

Use GitHub's [private vulnerability reporting form](https://github.com/SadPossum/GMA-Skeleton/security/advisories/new). Do not open a public issue for an undisclosed vulnerability and do not include credentials, personal data, or third-party confidential data in a report.

Include the affected repository and commit or release, the impact, reproducible steps or a minimal proof of concept, and any known mitigation. A composition issue may be reported here even when its root cause is in a mounted GMA repository; the report will be routed without requiring public disclosure.

We aim to acknowledge a complete report within three business days and provide an initial assessment within seven business days. These are response targets, not a contractual support SLA. Please coordinate public disclosure until a fix or mitigation is available.

There is currently no paid bug-bounty programme. Good-faith, non-destructive research against systems and data you own is welcome; denial of service, social engineering, persistence, and access to another person's data are out of scope.
